⚠️ Что делать, если кнопка «Восстановить» не работает?

Ручное восстановление через командную строку:

  1. Остановите сервер Django (Ctrl+C).
  2. Откройте терминал в папке проекта.
  3. Выполните команду, указав имя файла из папки backups/:
python manage.py restore_db db_backup_YYYYMMDD_HHMMSS.sqlite3

Например (самый свежий бэкап):

{% if backups %}python manage.py restore_db {{ backups.0.filename }}{% else %}python manage.py restore_db <имя_файла>.sqlite3{% endif %}

Все файлы бэкапов находятся в папке проекта:

backups/

Команда автоматически сохраняет страховую копию текущей базы перед заменой (файл before_restore_*.sqlite3 в той же папке). После восстановления перезапустите сервер Django.

Макросы быстрых действий

Каждый макрос показывает: что делает действие и какую команду выполнить.

1. Полный сброс dev-базы + админ
Удаляет локальную БД, прогоняет миграции и очищает данные. После flush автоматически создаётся админ aedgy.
del db.sqlite3
python manage.py migrate
python manage.py flush --noinput
2. Быстрое обновление вакансий
Запускает синхронизацию вакансий HH вручную.
python manage.py fetch_hh --pages 1
3. Открыть сервисные панели
Переход выполняется через горячие клавиши, без кнопок в модалке.
